1. HỆ THỐNG XÁC THỰC (Authentication System)
   --------------------------------------------
   Endpoint prefix: /api/v1/auth/

   [ĐĂNG KÝ]
   - POST /register
     • Đăng ký tài khoản mới (Nông dân hoặc Doanh nghiệp)
     • Nhận: fullName, email, phone, password, confirmPassword, role, agreeTerms
     • Validate: mật khẩu khớp, đồng ý điều khoản, email duy nhất
     • Tự động hash mật khẩu (bcrypt, salt 12)
     • Trả về: thông tin user + accessToken (cookie refreshToken)

   [ĐĂNG NHẬP]
   - POST /login
     • Đăng nhập bằng email hoặc số điện thoại + mật khẩu
     • Nhận: emailOrPhone, password, rememberMe
     • Thời gian cookie: 1 ngày (mặc định) / 30 ngày (rememberMe)
     • Bảo vệ brute-force: khóa tài khoản sau 5 lần thất bại, mở sau 15 phút
     • Cập nhật lastLogin sau đăng nhập thành công

   [ĐĂNG XUẤT]
   - POST /logout (yêu cầu đăng nhập)
     • Xóa refreshToken khỏi DB
     • Xóa cookie refreshToken

   [LÀM MỚI TOKEN]
   - POST /refresh-token
     • Dùng refreshToken (cookie hoặc body) để cấp accessToken mới
     • Tự động rotate refreshToken (cấp token mới mỗi lần refresh)

   [QUÊN MẬT KHẨU]
   - POST /forgot-password
     • Gửi token đặt lại mật khẩu (qua email - cơ sở hạ tầng)
     • Token hết hạn tự động

   [ĐẶT LẠI MẬT KHẨU]
   - POST /reset-password
     • Đặt lại mật khẩu bằng token từ email

   [CẬP NHẬT MẬT KHẨU]
   - PUT /update-password (yêu cầu đăng nhập)
     • Đổi mật khẩu với xác nhận mật khẩu hiện tại

   [XEM HỒ SƠ]
   - GET /me (yêu cầu đăng nhập)
     • Lấy thông tin người dùng hiện tại

   [CẬP NHẬT HỒ SƠ]
   - PUT /update-profile (yêu cầu đăng nhập)
     • Cập nhật: tên, số điện thoại, avatar

   [VÔ HIỆU HÓA TÀI KHOẢN]
   - DELETE /deactivate (yêu cầu đăng nhập)
     • Soft delete tài khoản (isActive = false)


2. MÔ HÌNH NGƯỜI DÙNG (User Model - MongoDB)
   -------------------------------------------
   Trường dữ liệu:
   - email (unique, lowercase)
   - password (hash bcrypt, ẩn khỏi response)
   - role: 'farmer' | 'enterprise'
   - firstName, lastName, fullName (tự động tạo)
   - phone (tùy chọn)
   - avatar (tùy chọn)
   - isActive (mặc định: true)
   - isVerified (mặc định: false)
   - refreshToken (ẩn khỏi response)
   - passwordResetToken + passwordResetExpires
   - passwordChangedAt, lastLogin
   - loginAttempts (đếm thất bại)
   - lockUntil (thời điểm mở khóa)

   Bảo mật:
   - Tự động hash mật khẩu trước khi lưu
   - Ẩn các trường nhạy cảm trong JSON response
   - Kiểm tra thay đổi mật khẩu sau thời điểm cấp JWT


3. API ROUTES NÔNG DÂN (Farmer Routes) - /api/v1/farmer/
   -------------------------------------------------------
   (Tất cả yêu cầu đăng nhập + role = 'farmer')
   - GET /dashboard   → Dữ liệu tổng quan nông dân (đang phát triển)
   - GET /crops       → Danh sách cây trồng / cánh đồng (đang phát triển)
   - GET /contracts   → Hợp đồng bao tiêu của nông dân (đang phát triển)
   - GET /orders      → Đơn hàng (đang phát triển)
   - GET /finances    → Dữ liệu tài chính (đang phát triển)


4. API ROUTES DOANH NGHIỆP (Enterprise Routes) - /api/v1/enterprise/
   -------------------------------------------------------------------
   (Tất cả yêu cầu đăng nhập + role = 'enterprise')
   - GET /dashboard  → Dữ liệu tổng quan doanh nghiệp (đang phát triển)
   - GET /suppliers  → Danh sách nhà cung cấp / nông dân (đang phát triển)
   - GET /contracts  → Hợp đồng bao tiêu (đang phát triển)
   - GET /warehouse  → Kho bãi / tồn kho (đang phát triển)
   - GET /analytics  → Phân tích thị trường (đang phát triển)


5. API SẢN PHẨM (Product Routes) - /api/v1/products/
   ----------------------------------------------------
   - GET  /          → Lấy tất cả sản phẩm (lọc, tìm kiếm) - đang phát triển
   - GET  /:id       → Chi tiết sản phẩm - đang phát triển
   - POST /          → Tạo sản phẩm mới (nông dân) - đang phát triển
   - PUT  /:id       → Sửa sản phẩm (chủ sở hữu) - đang phát triển
   - DELETE /:id     → Xóa sản phẩm (chủ sở hữu) - đang phát triển


6. MIDDLEWARE & KHẢ NĂNG KỸ THUẬT
   ----------------------------------
   - Bảo mật: Helmet (HTTP headers), CORS cấu hình theo môi trường
   - Nén dữ liệu: Compression middleware
   - Logging: Morgan (dev: 'dev', prod: 'combined')
   - Cookie Parser: httpOnly cookie cho refreshToken
   - Body Parser: giới hạn 10MB
   - Error Handler: xử lý lỗi toàn cục, phân loại AppError
   - Auth Middleware: JWT verify, restrictTo(role)
   - Validation Middleware: validate input đăng ký / đăng nhập / đổi mật khẩu
   - asyncHandler: bọc async/await để bắt lỗi tự động
   - Health Check: GET /health → kiểm tra trạng thái server

18. QUY TRÌNH TẠO HỢP ĐỒNG BAO TIÊU (/contract-flow)
    -------------------------------------------------------
    Yêu cầu đăng nhập (cả hai role)
    Luồng 5 bước có thanh tiến trình:

    Bước 1 - Đề xuất:
    - Tên sản phẩm (tự điền nếu đến từ trang sản phẩm)
    - Số lượng + đơn vị (tấn, kg, tạ)
    - Giá / đơn vị (nghìn VND)
    - Ngày giao hàng
    - Điều khoản thanh toán (50/50, 30/70, trả trước 100%, trả sau 100%)
    - Ghi chú

    Bước 2 - Xem xét:
    - Tóm tắt thông tin hợp đồng
    - Tổng giá trị hợp đồng (tự tính)
    - Phí hoa hồng PreOnic 3%

    Bước 3 - PreOnic Xác nhận:
    - Trạng thái xác minh các bên
    - Kiểm tra chất lượng sản phẩm
    - Bảo lãnh thanh toán

    Bước 4 - Ký hợp đồng:
    - Hiển thị điều khoản hợp đồng
    - Checkbox đồng ý điều khoản
    - Checkbox xác nhận qua PreOnic

    Bước 5 - Hoàn tất:
    - Thông báo hợp đồng đã ký
    - Số hợp đồng (PRE-2026-XXXX)
    - Hướng dẫn bước tiếp theo

    Auto-fill thông tin từ URL param ?product=<id>
    Validate từng bước trước khi tiến tiếp
